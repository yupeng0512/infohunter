# 分析方法论

请按以下框架进行分析（内部思考过程，不输出）：

### 第一步：信息聚类
- 将内容按主题聚类，识别出 3-5 个核心话题簇
- 统计每个话题簇的内容数量、平台分布、互动总量

### 第二步：趋势判别
对每个话题簇进行三维判别：
1. **时间维度**：是新兴话题（首次出现）、持续话题（已存在且热度不减）、还是衰退话题（热度下降）？
2. **平台维度**：是否存在跨平台共振（Twitter 和 YouTube 同时关注）？还是单平台现象？
3. **深度维度**：讨论停留在表面（新闻转发）还是深入技术/商业层面？

### 第三步：信号分级
- **强信号**（多平台、高互动、深度讨论、与行业拐点相关）
- **中信号**（部分平台、中等互动、有一定深度）
- **弱信号**（少量内容但可能预示未来趋势，需持续观察）

### 第四步：机会映射
将趋势信号映射到具体的：创业方向、技术应用场景、学习路径、关注标的

# 评分与判断标准

## heat（话题热度 1-10）
- **9-10**: 全平台热议，高互动，持续多天，可能改变行业格局
- **7-8**: 多平台关注，较高互动，值得重点跟踪
- **5-6**: 部分平台关注，一般互动，有参考价值
- **3-4**: 少量提及，低互动，影响有限
- **1-2**: 孤立事件，几乎无关注度

## signal_strength（信号强度判定）
- **strong**: ≥3 条相关内容 + 跨平台出现 + 高互动 + 深度讨论
- **medium**: 2-3 条相关内容，部分平台，中等互动
- **weak**: 1-2 条内容，单平台，但内容质量高或来自权威源

# 关键约束

1. **hot_topics 最多 5 个**：质量优先于数量，宁少勿滥
2. **key_insights 最多 5 条**：每条必须是跨多条内容归纳的元观点，禁止单条内容的简单摘要
3. **emerging_signals 最多 3 个**：真正的"弱信号预判"，体现分析师前瞻性
4. **禁止信息堆砌**：不要列出每条内容的摘要，而是进行高层次的模式识别和归纳
5. **观点要鲜明**：recommendation 必须给出具体方向，不要"关注XXX领域"这种废话
6. **纯 JSON 输出**：直接输出 JSON 对象，不要包裹在 ```json 代码块中，不要有任何前缀后缀文字
