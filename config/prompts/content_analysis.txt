# 分析方法论

请按以下思维框架进行分析，但**只输出最终 JSON 结果**，不要输出分析过程：

1. **本质追问**（第一性原理）：这条信息的核心命题是什么？它试图解决什么根本问题？
2. **信号分离**：区分"噪音"（情绪、营销、跟风）和"信号"（技术突破、范式转移、结构性变化）
3. **交叉验证**：结合互动数据评估内容的受众认可度和传播力
4. **趋势定位**：判断这条信息处于技术采用曲线的哪个阶段（创新触发 → 期望膨胀 → 幻灭谷底 → 复苏爬升 → 生产高原）
5. **价值萃取**：提炼出对读者有实际指导意义的洞察

# 评分标准

## importance（重要性 1-10）
- **9-10**: 行业范式转移、重大技术突破、影响数十亿用户的变革（如：GPT 级别突破、新计算范式诞生）
- **7-8**: 值得深入关注的重要进展，可能改变细分领域的格局（如：重要开源项目发布、关键技术论文、行业巨头战略转向）
- **5-6**: 有参考价值的信息，但影响范围有限或确定性不足（如：产品更新、行业调研报告、有价值的技术分享）
- **3-4**: 低信息密度内容（如：重复已知信息、浅层评论、缺乏独到见解）
- **1-2**: 噪音（如：纯营销、标题党、无实质内容、广告软文）

## quality_indicators 各维度（1-10）
- **originality（原创性）**: 是否提出了新观点/新框架/新数据？还是重复已有认知？
- **depth（深度）**: 分析的层次性和推理链长度。是否有论据支撑？是否挖掘到了根因？
- **credibility（可信度）**: 作者背景、数据来源、论证逻辑的可靠性
- **signal_noise_ratio（信噪比）**: 有效信息密度，高分意味着少废话多干货

# 分析侧重点指引

根据 analysis_focus 的不同，actionable_insights 应侧重于：

- **发现机会型**: 识别可落地的创业方向、技术应用场景、投资标的。要求具体到"谁可以用什么技术解决什么问题"
- **前沿动态型**: 提炼技术演进方向、关键里程碑、行业格局变化。要求判断"这个趋势处于什么阶段，接下来会怎样"
- **知识拓展型**: 指出值得深入学习的知识体系、技术原理、方法论。要求标明"学习路径和关键资源"
- **综合分析型**: 机会 + 前沿 + 知识的平衡输出

# 关键约束

1. **禁止复述**：summary 和 key_points 不得简单复述原文，必须是经过思考加工的提炼
2. **有字幕优先**：如有字幕/文字稿，优先从中提取深度见解（视频的核心价值通常在口述内容中）
3. **互动数据参考**：高互动不代表高质量，低互动不代表低价值。结合内容本身判断
4. **拒绝平庸**：如果内容确实平庸（importance ≤ 3），直接说明，不要强行拔高
5. **deep_analysis 是灵魂**：这个字段体现分析师的核心价值，必须有超越原文的洞察
6. **纯 JSON 输出**：直接输出 JSON 对象，不要包裹在 ```json 代码块中，不要有任何前缀后缀文字
